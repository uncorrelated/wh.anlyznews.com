はじめに

    • タブ切り替え（{.tabset}）は動かない
    • H1タグ（#）と脚注も使えない
  • 調べるのが手間かも知れないところをまとめる
    • マークダウン記法自体の話はネット上に大量に資料がある
    • CSSまわりの話はR Markdownのバージョンが上がればバッドノウハウになる可能性大
  • 日本に3人ぐらいエヴァンジェリストがいて便利なパッケージを紹介しているが、ベタな扱い方を模索
  • R Markdown Cheat Sheetに書いてある話もある
  • R Markdown: The Definitive Guideに大まかなknow-howはある

表紙のためのスタイルシート

cssファイルを指定しないと、

  • タイトル文字列のサイズ縮小
  • タイトル画像の拡大が

ができないので、ヘッダーに以下のような指定がいる。

output:
    ioslides_presentation:
        css: styles.css
        logo: 'icon.png'

文字入力

HTMLのタグとタグの間は、HTMLとして解釈される

「強敵（トモ）」と入力するには、<ruby>強敵<rp>（</rp><rt>トモ</rt><rp>）</rp></ruby>

特殊文字の入力もHTMLと同様に行なえる

「✓ ♨ ♡ ⚑ ① ② ③」と入力するには、&#10003; &#x2668; &#x2661; &#9873; &#x2460; &#x2461; &#x2462;

エスケープはC言語の文字列などと同様

文字の前に\を書くとエスケープできて、特殊文字もそのまま表示できる。

TeX形式の数式の入力

$x^2$と$で挟むとインラインで\(x^2\)と表示できる。 $x=`r sum(1:10)`$とインラインコードを数式内に挿入すると、\(x=55\)と計算され表示される。

$$
y = C_1 e^{-\frac{k}{2} t} \cos{ \Bigg ( t \sqrt{ \omega^2 - \frac{k^2}{4}} \Bigg )} + 
C_2 e^{-\frac{k}{2}t} \sin{ \Bigg ( t \sqrt{ \omega^2 - \frac{k^2}{4}} \Bigg ) }
$$

とブロック要素で書くと、

\[ y = C_1 e^{-\frac{k}{2} t} \cos{ \Bigg ( t \sqrt{ \omega^2 - \frac{k^2}{4}} \Bigg )} + C_2 e^{-\frac{k}{2}t} \sin{ \Bigg ( t \sqrt{ \omega^2 - \frac{k^2}{4}} \Bigg ) } \]

と表示できる。

行頭に>を置くな，<pre>と</pre>で挟め

マークダウン記法では行頭の>で引用になるが、ioslides_presentationでははっきりしない表示になるので、styles.cssに以下のように定義して<pre>と</pre>で挟む方が分かりやすくなる。

pre {
    color: #FFFFFF;
    font-weight:bold;
    background-color: #000000;
}

表組みの色の変更

デフォルトのパイプ表の色が独特なので、CSSをいじって変更しよう。

table.rmdtable tbody tr.header th {
    background: #121212; color: #FFFFFF;
}

table.rmdtable tbody tr.even {
    background-color: #FFFFFF;
}

table.rmdtable tbody tr.odd {
    background-color: #FFF0F0;
}

table.rmdtable tbody tr td {
    color: #000000;
}

交差検証（LOOCV）の表示①

自動で表が作られない場合、例えば交差検証（LOOCV）は、

library(boot)
frml <- c("y ~ x + z", "y ~ x", "y ~ z")
delta <- matrix(NA, length(frml), 2)
rownames(delta) <- frml
colnames(delta) <- c("raw", "adjusted")
for(i in 1:length(frml)){
    delta[i, ] <- cv.glm(df01, glm(formula(frml[i]), data=df01))$delta
}

パイプ表の中にTeXの数式とインラインコードを入れて表にする。

プロットの表示

code chunkの中にplot命令を入れたらよいだけだが、code chunkのオプションはfig.align=“center”, fig.width=9, fig.heigth=8のようにつけ、Rのコードでもpar(mar=c(4, 4, 0.1, 0.1))などと表示調整した方がよい。

終わり

  • 自由度は低く、見出しと図表があればプレゼンできる人向けで、官公庁パワポを作りたい人は避けるべし
  • モバイル非対応と言うか、ケータイで見ると画面全体を見るのが困難
  • CSSをいじらないといけない所が多い
    • 画像背景にあわせてH2見出しのフォント色を変えるのはspanタグでよいのか？
    • 箇条書きの文字色などは、##に{}で指定したCSS IDで変更できるので、気持ちが悪い
  • 頻繁に定型レポートを書く人にはたぶん便利
  • rmarkdown::renderでRmdからhtmlを生成できるので自動化もできる
  • code chunkにbrowserを埋め込めるのでデバッグもできるが、テスト済みコードを埋めよう
